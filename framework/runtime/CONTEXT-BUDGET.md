# Context Budget

## 文档定位

本文只承接运行时成本的量化接口。成本约束如何改变边界、路由与验证的适用条件，属于更高层分析，留在白皮书论述，不在本文件展开。

本文不定义成本的经济学模型，不提供固定阈值表，不预测 provider 费率变化。

---

## 预算对象

四类可量化消耗：

| 对象 | 计量方式 |
|------|----------|
| **模型推理资源（compute footprint）** | 运行时模型侧资源约束的抽象对象，可由 token、上下文长度、模型档位或 provider 计量单位实例化 |
| **工具调用** | 外部工具/API 的调用次数与响应负载 |
| **时间** | 单次任务或单轮推理的 wall-clock 时长 |
| **人类审阅注意力** | 需要人类确认、复核或介入决断的动作次数 |

---

## 预算入口

三类实例化位置，由约束强到弱：

| 入口 | 说明 |
|------|------|
| **系统级硬上限** | 节点全局不可突破的预算封顶 |
| **角色契约默认值** | `ROLE-CONTRACT.md` 中该角色的常规预算配置 |
| **单次任务覆盖值** | 当前任务开始时显式声明的临时覆盖 |

同一对象的多级入口冲突时，以最小值生效。此规则仅为约束解释原则，不预设具体的配置加载机制或运行时实现。

---

## 默认动作

超预算或预算不可查时的标准动作：

| 场景 | 动作 |
|------|------|
| 预算余量不足支撑当前展开 | 强制收敛：停止继续展开，输出当前已积累的判断 |
| 预算不可查（未声明入口） | 默认拒绝：暂停任务，显式说明缺失项 |
| 超出角色契约默认值但未触硬上限 | 暂停并显式声明需人类决断 |

---

## 接口映射

本文定义的预算对象与入口，供 framework 各模块在以下场景中回查：

### 任务展开判断

`OPERATING-RULES.md` 执行收敛评估前，回查当前任务的预算余量。余量不足时，不进入新展开，触发收敛或暂停并显式声明需人类决断。

### 路由扩展限制

`ROLE-CONTRACT.md` 在定义「可直接承担的判断范围」时，可将本文件预算对象作为可选约束参数回查。超出预算参数的路径标记为不可选，但不改变角色本身的静态收口资格。

### 验证深度影响

`TRUTH-CONTRACT.md` 在决定验证标准时，回查验证相关预算对象的可用余量。成本不可承受时，标记 `[unverified]` 并说明预算约束，不降格为伪完成。

### 记忆写回影响

`CORRECTION-WRITEBACK.md` 判断纠错内容是否写入长期记忆时，回查模型推理资源 / 时间相关预算对象的可用余量。超支时降级为摘要写回或延后写回，不直接丢弃。
