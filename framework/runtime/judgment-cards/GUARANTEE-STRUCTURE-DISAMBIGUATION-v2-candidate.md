# GUARANTEE-STRUCTURE-DISAMBIGUATION v2 candidate

> v2 candidate，用于替换讨论前的并行观察。旧卡暂不作废。

---

### 1. 卡片用途

本卡用于**预判断**：在信息不完备时完成结构层拆分，标记缺口，明确是否可进入最终判断。

不是法律定性工具，不替代最终法律意见。

---

### 2. 适用场景

- 同时出现"保函""风险参与""反担保""转开"等术语
- 易把"转开保函+反担保"与"不出资风险参与"混为一谈

---

### 3. 起手检查

| 检查项 | 通过标准 | 未通过动作 |
|--------|---------|-----------|
| 请求方身份 | 明确 | 询问后重试 |
| 我能代表谁 | 结构分析层 | 若被要求法律定性，升级 |
| 基础文本充分 | 有协议文本 | 暂停，标记缺文本 |
| 输出使用场景 | 明确是否被第三方依赖 | 若不确定，fail-closed |

**任一未通过** → 按对应动作处理，不强行继续

---

### 4. 结构判断主轴（三维拆分）

| 维度 | 关键问题 |
|------|---------|
| 担保关系 | 是否存在担保链结构 |
| 合同性风险承担 | 是否为合同性风险分担安排 |
| 是否出资 | 是否涉及资金实际流转 |

**拆分后** → 进入输出选择

---

### 5. 预判断输出

当基础文本不明、位置不足时：

```
【预判断】
- 结构拆分：（三维结果）
- 信息缺口：（具体缺什么）
- 建议：（向谁/如何补足）
```

**禁止**：写成"我认为..."或"这两者本质不同"

---

### 6. 最终判断输出

当信息充分、位置适当时：

```
【最终判断】（信息来源：verified/inferred）
- 结论：
- 三维论证：
- 风险提示：
```

---

### 7. 失败协议

| 触发条件 | 当前动作 | 恢复条件 |
|---------|---------|---------|
| 基础文本缺失关键条款 | 暂停 | 获得完整文本 |
| 被要求法律定性 | 升级 | 移交法律位置 |
| 当事人意图不明 | 暂停 | 获得书面确认 |
| 输出使用场景不明 | fail-closed | 人工评估后决定 |

**fail-closed 不是不判断，是判断此刻不能由我完成**

---

### 8. 写回建议

按判断类型标记：

| 类型 | 标记 |
|------|------|
| 预判断 | [PRE] |
| 最终判断 | [FINAL] |
| 暂停 | [PENDING] |
| fail-closed | [CLOSED] |
