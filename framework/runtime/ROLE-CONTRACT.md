# 角色契约

> 专业任务角色边界接口。把 identity 收窄成可执行的工作角色。

---

## 文档定位

- `IDENTITY.md` 回答"我是谁"（稳定、长期）
- `ROLE-CONTRACT.md` 回答"我在这个专业域里承担什么判断"（灵活、短期）

它不是 soul，不是 memory，不是 governance checklist。它是角色边界接口。

---

## 一、角色契约的目的

**Identity 不足以承担专业任务边界的原因**：
- Identity 描述节点本体，不随任务变化
- 专业任务需要明确的判断权限与责任边界
- 同一 identity 在不同任务中可能承担不同角色

**为什么需要 role contract**：
- 收窄泛化能力，聚焦当前任务域
- 明确可判断与不可判断的边界
- 建立人类纠错的入口与优先级

**在通用框架中的位置**：
- 位于 Role 模块
- 承接 Constitution 的方向约束
- 为 Runtime 提供操作边界

---

## 二、角色契约最小字段

| 字段 | 说明 |
|------|------|
| **角色名称** | 当前任务中的角色标识，如"代码审查员"、"数据分析师" |
| **角色目标** | 本角色的核心产出目标，一句话可验收 |
| **可直接承担的判断范围** | 本角色可直接做出专业判断的领域清单 |
| **不直接承担的判断范围** | 超出本角色权限、必须升级或移交的判断领域 |
| **待验证内容的处理方式** | 遇到未验证信息时的标准动作（标注/暂停/升级） |
| **高风险口径升级规则** | 何种情况下必须停止自主判断、升级给人类 |
| **人类纠错优先级** | 当人类给出纠正时，哪些必须立即执行、哪些可讨论 |
| **可写回内容** | 本角色产生的哪些经验/规则可以写回长期记忆 |
| **不可直接写回内容** | 哪些判断必须经人类确认后才能写回 |

---

## 三、角色契约的元动作对应

| 元动作 | 在本文件中的体现 |
|--------|------------------|
| **收窄** | 通过"可直接承担的判断范围"和"不直接承担的判断范围"明确边界 |
| **升级** | 通过"高风险口径升级规则"定义何时停止自主、交还人类 |
| **接受纠正** | 通过"人类纠错优先级"明确纠正的响应方式与速度 |
| **口径分流** | 通过"待验证内容的处理方式"区分已验证/未验证/待确认的内容 |

---

## 四、与其他文件的关系

| 文件 | 关系 |
|------|------|
| `IDENTITY.md` | Identity 是本体，Role Contract 是身份在特定任务中的收窄表达。Identity 不变，Role Contract 随任务变。 |
| `USER.md` | User 是协作者信息，Role Contract 中的"人类纠错优先级"和"升级规则"需要知道向谁升级。 |
| `MEMORY-INDEX.md` | Memory Index 提供历史信息，Role Contract 决定在本任务中如何择用这些信息。 |
| `judgment-cards/` | Judgment Cards 是可复用判断模板，Role Contract 明确在本任务中调用哪些卡片、不调用哪些。 |
| `CORRECTION-WRITEBACK.md` | Correction Writeback 是纠错写回记录，Role Contract 中的"可写回内容"字段决定什么可以进入该文件。 |
| `TRUTH-CONTRACT.md` | Truth Contract 是真实性协议，Role Contract 是真实性协议在特定角色上的具体化（什么该说、什么不该说）。 |

## 五、建议模板

```markdown
# Role Contract: [角色名称]

## 生效信息
- 任务 ID：
- 生效时间：
- 预期终止：

## 角色目标
[一句话可验收的产出目标]

## 判断范围

### 可直接承担
- [ ] 判断项 1
- [ ] 判断项 2

### 不直接承担（必须升级）
- [ ] 判断项 A
- [ ] 判断项 B

## 待验证内容处理
- 标注方式：[待验证]
- 默认动作：暂停/继续/升级

## 高风险升级规则
- 遇到 X 情况 → 立即升级
- 遇到 Y 情况 → 先标注再等待确认

## 人类纠错优先级
- P0（立即执行）：安全、合规、事实错误
- P1（讨论后执行）：风格、偏好、优化建议
- P2（记录不执行）：超出当前角色范围

## 写回规则

### 可直接写回
- 成功模式
- 失败分型

### 需确认后写回
- 边界调整
- 规则变更
```

---

## 六、边界声明

- **角色契约不是完整 prompt**：它只定义边界，不定义具体执行步骤
- **角色契约不替代真实性合同**：它细化 Truth Contract 在特定角色上的应用，但不取代真实性约束
- **角色契约不等于技能说明书**：技能说明"怎么做"，角色契约说明"什么能做、什么必须升级"
