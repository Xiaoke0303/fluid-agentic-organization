# FAO (Fluid Agentic Organization)

面向 AI 时代的最小组织框架，关注人与代理在任务协作中的授权、边界、留痕与担责。

---

## 入口

- `toolkit/minimal-core/`  
  长期运行节点的最小内核：方向、记忆、节律

- `toolkit/governance/`  
  真实性治理：边界暴露、外部调用检查、失败报告、最小自主排查

- `templates/`  
  节点接入模板：身份锚点、用户接口占位

- `WHITEPAPER.md`  
  五层模型与必要条件

minimal-core 解决长期运行中的不漂移，governance 解决对外交互中的不伪装。

---

## 为什么这个问题现在重要

- AI 正在从工具变成可执行任务的代理
- 但大多数组织仍缺少授权、边界、责任与追溯机制
- 关注点不是让 AI 更聪明，而是让协作关系更清楚

如果你已经在让 AI 做原本需要人来授权、协调或承担后果的事，这个实验可能与你有关。

---

## 最小任务单元示意

| 要素 | 内容 |
|---|---|
| 任务 | 起草对外回复一项客户争议事项 |
| AI 代理 | 整理事实、归纳争议点、生成回复草稿 |
| 人类锚点 | 判断口径、决定是否发送、承担后果 |
| 边界 | AI 不可单独对外承诺或发送 |
| 追溯 | 草稿、修订、发送决策均留痕 |
| 生命周期 | 该事项处理完成即解散，记录保留 |
