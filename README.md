# FAO (Fluid Agentic Organization)

面向 AI 时代的最小组织框架，关注异质主体（人类与智能体）在任务协作中的授权、边界、留痕与担责。

---

## 快速开始

**FAO 不是抽象层级图，而是落实在具体任务中的责任安排。**

| 层级 | 用途 |
|------|------|
| `toolkit/minimal-core/` | 长期运行节点的最小内核：方向、记忆、节律 |
| `toolkit/governance/` | 真实性治理：边界暴露、外部调用检查、失败报告 |
| `templates/` | 节点接入模板：身份锚点、用户接口占位 |
| `whitepaper/` | 异质主体协作条件下的边界、路由与治理 |

---

## 为什么这个问题现在重要

- AI 正在从工具变成可执行任务的代理
- 但大多数组织仍缺少授权、边界、责任与追溯机制
- 关注点不是让 AI 更聪明，而是让协作关系更清楚

如果你已经在让 AI 做原本需要人来授权、协调或承担后果的事，这个实验可能与你有关。

---

## 最小任务单元示意

**FAO 不是抽象层级图，而是落实在具体任务中的责任安排。**

| 要素 | 内容 |
|---|---|
| 任务 | 起草对外回复一项客户争议事项 |
| AI 代理 | 整理事实、归纳争议点、生成回复草稿 |
| 人类锚点 | 判断口径、决定是否发送、承担后果 |
| 边界 | AI 不可单独对外承诺或发送 |
| 追溯 | 草稿、修订、发送决策均留痕 |
| 生命周期 | 该事项处理完成即解散，记录保留 |
