# FAO Minimal Core

一个实验性的最小骨架，帮助长期运行的 agent system 保持方向、连续性与节律。

---

## 受众

本工具包默认读者已经接触过长期运行的 agent system，或至少对工作目录、配置文件、记忆分层等概念有基本感觉。它不是零基础入门材料。

---

## 它解决什么

长期运行的 agent 容易出现的三种失稳：

- **方向漂移**：运行越久，原始意图越模糊
- **记忆膨胀**：长期记忆变成杂物堆，查询成本上升，关键信息淹没
- **节律失控**：要么过度反应，要么在无人区空转

Minimal Core 用三个薄层应对：

| 文件 | 作用 |
|------|------|
| soul.md | 稳定方向，知道"不做什么" |
| memory.md | 保持连续，但不让记忆成为负担 |
| heartbeat.md | 建立节律，定期止观与代谢 |

---

## 它不解决什么

- **具体任务执行** —— 交给 TASK.md 或项目级文档
- **个性化矫正** —— 如去魅检查、贴身语境，需各节点自行添加
- **技术实现** —— 不规定存储格式、同步机制、版本控制
- **组织协作** —— 不规定多人如何共用、如何审计、如何分支

---

## 最小使用方式

这三份文件可以作为长期运行 agent system 的最小内核：

1. `soul.md` 负责方向
2. `memory.md` 负责长期连续性
3. `heartbeat.md` 负责暂停、清理与自检的节律

它们既可以单独使用，也可以作为更大系统中的基础层。

---

## 不要把它当成

- 完整系统模板
- 最终最佳实践
- 陪伴型人格包
- 可直接上线的生产标准

---

## 名称不是重点

soul / memory / heartbeat 这些名称不是重点。重点是：

- 一个层负责方向
- 一个层负责长期连续性
- 一个层负责节律、暂停与清理

可以改名，但不应把这些功能重新混成一团。

---

## 为什么只保留这三个

当前实验中的一个工作判断是：长期失稳的 agent，问题往往不只出在工具能力，也出在方向、记忆与节律。

更多层可以往上加，但这三层是基底。

## 文件说明

**soul.md**  
最小方向层。只写"稳定朝向"，不写长篇哲学，不写个体叙事。

**memory.md**  
最小连续层。只写"什么该记住、什么不该"，不写具体偏好，不写项目细节。

**heartbeat.md**  
最小节律层。只写"何时暂停、检查、清理"，不写运行脚本，不写具体检查项。
