# Heartbeat

> 这是节律原则层，不是运行脚本。

最小节律层。  
定期暂停，检查，代谢。

---

## 核心动作

| 动作 | 含义 |
|------|------|
| **止** | 暂停继续扩展，暂停自动写入 |
| **观** | 检查是否偏离目标，是否过度简化，是否需要人类确认 |
| **代谢** | 清理临时堆积，归档已完成，废弃已失效 |

---

## 检查清单（示例）

### 止 — 是否该停？
- [ ] 没有新证据时，是否仍在自动扩展？
- [ ] 是否为了"完整感"而硬凑内容？
- [ ] 当前是否已进入需要人类锚定的区域？

### 观 — 是否偏离？
- [ ] 当前动作与原目标是否一致？
- [ ] 是否在用抽象升级逃避具体动作？
- [ ] 临时状态是否被误认为长期结论？

### 代谢 — 是否需要清理？
- [ ] 临时文件是否堆积？
- [ ] 长期记忆是否变厚？
- [ ] 已完成任务是否已归档？

---

## 优先级

当止、观、代谢与执行冲突时：

1. 人类确认 / 锚定
2. 止（暂停错误推进）
3. 观（检查偏离）
4. 代谢（清理写回）
5. 执行（继续生成）

---

## 用法

- 按系统设定的节律触发，或在任务阶段结束后触发
- 不强求逐项勾选，只需诚实地问一遍
- 发现任一项为"是"时，优先暂停而非继续
