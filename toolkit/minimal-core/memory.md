# Memory

> 这是长期记忆原则层，不是日志仓库。

最小连续层。  
长期记忆应该薄，薄到可以一眼看完。

---

## 原则

1. **只保留会影响未来判断的信息**
   - 偏好、约束、关键决策 → 保留
   - 临时状态、情绪碎片、单次对话细节 → 不保留

2. **临时判断不直接进入长期记忆**
   - 新想法先进入待验证区（如 incubation.md）
   - 通常在多个任务或讨论中重复出现，且能稳定表达后，方可合并

3. **外部查询中的未验证判断，不直接进入长期记忆**
   - 猜测、失败后的估计、临时推断 → 不写入主干
   - 二手转述若验证状态不清 → 不作为稳定记忆沉淀
   - 只有跨任务重复成立、且证据来源稳定的结论 → 可进入较稳定记忆

4. **允许修订与废弃**
   - 记忆不是累积，是策展
   - 过时信息标记为废弃，不强行解释其"历史价值"

5. **保持一句能说清**
   - 如果长期记忆厚到需要索引，它已经太厚
   - 记忆不是为了保留完整过程，而是为了保留真正可靠、会影响未来判断的部分

---

## 结构建议

```
memory/
├── memory.md       # 主干：稳定规则与长期主题
├── incubation.md   # 分支：待验证的新判断
└── changelog.md    # 修订记录：何时改了什么
```

---

## 检查项

- [ ] 本次要写入的内容，真的会影响未来判断吗？
- [ ] 它有稳定的证据来源吗（不是猜测或单次观察）？
- [ ] 它能用一句话稳定表达吗？
- [ ] 三个月后再看，它仍然有价值吗？
