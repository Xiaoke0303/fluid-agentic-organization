# External Call Checklist

> Status: legacy source  
> Current framework-native version: `../../framework/runtime/EXTERNAL-CALL-PROTOCOL.md`  
> Note: This file is retained for historical traceability. For current runtime use, prefer the framework/runtime version.

> 执行外部操作前的自检清单。

适用于：网页访问、文件读取、API 调用、环境检查、代码执行、浏览器操作、其他外部动作。

---

## 前置检查

- [ ] **已声明能力边界** — 明确说明本次操作的范围和限制
- [ ] **已确认工具/权限** — 确认自己真的有该工具，权限足够

---

## 执行记录

- [ ] **已记录目标对象** — URL / 文件路径 / API 端点 / 环境变量名
- [ ] **已记录实际动作** — 具体调用了什么，输入是什么
- [ ] **已记录时间戳** — 何时执行

---

## 结果与证据

- [ ] **已记录原始结果** — 返回了什么（成功/失败/超时/错误码）
- [ ] **已记录失败点** — 若失败，具体哪一步出错
- [ ] **已保留证据链** — 工具名、目标、输入、输出、时间戳齐全

---

## 失败后排查

- [ ] **已完成最小自主排查** — 路径/大小写/目录/线索/替代入口
- [ ] **已明确区分** — 已验证内容 vs 推断内容

---

## 完成态措辞检查

**只有满足以下条件，才允许使用"已查询 / 已确认 / 已验证"：**

- [ ] 实际执行了工具调用
- [ ] 实际获取到了结果
- [ ] 可以提供完整的证据链（工具、目标、输入、输出、时间）
- [ ] 结果经过检查，不是错误页面/超时/空响应

**若不满足以上条件，必须使用以下措辞：**
- "尝试访问，但..."（说明失败点）
- "无法确认，因为..."（说明缺失的能力或条件）
- "基于...推断"（明确标注这是推断，不是验证）

---

## 快速判断

| 情境 | 正确措辞 |
|------|----------|
| 成功获取且有证据 | "已查询：..." + 五项记录 |
| 尝试失败 | "尝试访问失败：..." + 失败点 + 缺失项 |
| 无法访问 | "无法确认：..." + 阻塞点 + 继续条件 |
| 部分成功 | "已获取 A，但 B 失败：..." |
