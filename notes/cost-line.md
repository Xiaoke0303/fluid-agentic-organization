# 成本主线工作笔记（草案）
## 能量 / 信息 / 空间 / 时间

### 1. 为什么现在要补这条线

FAO 目前已经解释了几个关键问题：为什么任务会流动，为什么责任未必随之流动，为什么组织需要边界、路由与治理。

但如果继续往前问，还会遇到另一类问题：为什么某些路由会长期胜出，为什么某些岗位会被拆解、压缩甚至消失，为什么同样都能完成任务，不同组织形态最后呈现出的稳定性和生命力会差这么多。

这些问题，单靠"主体—边界—路由—治理"还不够。这里缺少的一条线，是**成本**。

成本不是任务完成之后的会计结算，也不是"降本增效"的口号。它更像一种现实约束：决定哪些路径会被采用，哪些岗位会被重组，哪些组织形态能够持续。

因此，成本不是 FAO 的附属变量，而应当被看作一条尚未展开、但已经越来越重要的主线。

### 2. 为什么不用"物质"

如果从第一性原理压缩，FAO 当前讨论的并不是一般意义上的世界构成，而是：**一个任务在组织中流动时，究竟受哪些最基本的约束。**

在这个层面上，"物质"当然重要，但它在当前讨论里往往并不作为独立的强约束出现。它通常已经被空间分布、能量消耗和时间延迟吸收。

为了避免概念重叠，当前更适合先用四个基础维度来理解任务流动：

- 能量
- 信息
- 空间
- 时间

这不是否认物质的重要性，而是为了让分析更干净。奥卡姆剃刀在这里的作用，不是删掉现实，而是删掉重复命名。

### 3. 四个基础约束

#### 3.1 能量

能量回答的是：**这个任务要消耗什么资源。**

在当前语境下，它不只是狭义的电力，而是所有完成任务所需的消耗能力。它可以表现为人力投入，也可以表现为算力、token、执行成本、维护成本。

即便一条任务路径在能力上可行，也不一定在资源上可持续。能量维度提醒我们：组织判断不能只问"能不能做"，还要问"值不值得这样做"。

#### 3.2 信息

信息回答的是：**这个任务要处理多少不确定性。**

很多所谓"工作"，其核心其实是降低不确定性：检索、判断、校验、消歧、记忆、写回，本质上都在处理信息。

从这个角度看，framework 里已经长出来的大量接口，其实都属于信息维度：真实性合同、判断卡片、纠错写回、薄记忆索引，本质上都是在管理不确定性如何被处理、保留和再利用。

因此，信息不是材料堆积，而是**降低不确定性的处理能力**。

#### 3.3 空间

空间回答的是：**这个任务跨越哪些主体、边界与接口。**

这里的空间不是单纯的物理地理，而是分布结构。人和 agent 是否在同一节点，判断与执行是否在同一主体，责任链是否跨部门、跨系统、跨组织，这些都属于空间问题。

很多组织摩擦，看起来像"人不够""工具不行"，其实是空间分布出了问题：接口太多，传递太长，责任太散，导致协调成本迅速上升。

所以空间维度真正关心的，是**任务、判断和责任如何分布**。

#### 3.4 时间

时间回答的是：**这个任务在什么顺序、延迟与持续中展开。**

一个任务不是静止的点，而是一条链。前置检查、升级等待、返工、纠错、记忆沉淀、岗位演化，全部都发生在时间里。

时间维度还意味着：判断不会一次性完成，而会随着新证据到来不断更新。在这个意义上，**贝叶斯更新可以被理解为时间中的信息修正机制**。它不是第五个维度，而是说明：主体如何在时间中根据新增证据调整自己的后验判断。

因此，时间不是钟表，而是**顺序、延迟、持续与演化**；而贝叶斯则是这条时间线上的一种最小更新逻辑。

### 4. 成本是什么

如果能量、信息、空间、时间是四个基础约束，那么成本就不是第五个基础维度。

更准确地说：

**成本是这四类约束在现实组织中的价格表达。**

它至少可以暂时分成四类：

- 人力成本：判断能力与时间投入的价格
- token / 算力成本：信息处理与能量消耗的价格
- 协调成本：空间分布、接口摩擦、责任传递的价格
- 时延成本：等待、升级、返工与顺序安排的价格

这样理解的好处是，成本不会悬浮成一个独立口号。它直接从四个基础约束中长出来，成为这些约束在现实世界里的可感、可比、可筛选形式。

### 5. 成本如何进入 FAO

#### 5.1 进入边界

边界不只是责任边界，也会逐渐表现为成本边界。某些任务之所以不再由一个完整岗位整体承担，不一定是因为理论上分工更优，而是因为成本结构已经逼着它被拆成更小接口。

也就是说，边界并不只由规范塑造，也会被成本重写。

#### 5.2 进入路由

路由不只回答"谁能做"，还回答"谁做更便宜、更快、更稳"。同一个任务，原来可能走完整专家路径，后来变成 agent 初筛、人类收口，并不一定是因为理念进步，而是因为总成本曲线发生了变化。

因此，长期胜出的路由，往往不是最强的路由，而是**责任还能守住、总成本又更低的路由**。

#### 5.3 进入治理

治理不是没有代价的。让责任可见、失败可见、验证状态可见，本身就需要支付协调、记录、校验与维护成本。

因此，治理也不能只谈规范正确，还要问：为了让组织不失真，这套治理要付出什么成本，它是否可持续。

#### 5.4 进入生态位

组织形态之间的竞争，最后不会只表现为"谁更聪明"，而更可能表现为：**谁既守住责任，又把总成本压得更低。**

从这个意义上说，成本线一旦进入 FAO，FAO 就不只是在解释任务协作，还开始解释组织演化：为什么某些岗位会消失，为什么某些结构会扩张，为什么某些协作方式会占据生态位。

### 6. 对岗位变化的一个判断

真正消失的往往不是整份工作，而是其中那些已经可以被低成本接口接管的部分。

岗位通常不是一下子消失，而是先被拆解、再被重组。表面上看是裁员，冰山下面往往是：判断被抽出、执行被标准化、协调被平台化、部分职责被 agent 化，最后才体现为岗位数量和岗位名称的变化。

所以成本线真正解释的，不只是"企业要不要裁员"，而是**一个岗位为何会被重新定价、重新拆分、重新组织**。

### 7. 当前不做什么

这条线虽然重要，但当前还不适合直接硬并进 framework 的每个文件。至少在当前阶段，先不做以下几件事：

- 不把成本写成 framework 的第五层或额外控制面
- 不把成本变成"降本增效"的口号化表达
- 不急着回写白皮书每一章
- 不把这份工作笔记当成最终理论定稿

当前更合适的做法，是先把它当作一条 side note：独立站住，再决定如何进入白皮书主文。

### 8. 当前收口

FAO 不只是主体与责任的问题，也是资源消耗与不确定性处理的问题。任务路由不仅受边界约束，也受能量、信息、空间、时间四类约束共同塑形。成本则是这些约束在现实组织中的价格表达。

因此，成本不是 FAO 的补充变量，而是决定路由能否持续、岗位如何重组、组织形态如何胜出的现实约束。
