# 案例观察卡：傅盛 / 三万

- **观察对象**：傅盛（猎豹移动董事长兼CEO）与其AI Agent "三万"
- **时间范围**：2026年2月23日—3月9日（公开观察期，约14天）
- **主要公开来源**：
  - 傅盛直播复盘（20万+观看）
  - 极客公园报道《从OpenClaw到EasyClaw》
  - 北京商报《"龙虾"生意经》
  - 观察者网用户投稿《AI Agent的「最后一公里」》

## 3. 当前可见的权限边界

**Agent 做了：**
- 实际执行拜年消息发送（611条，4分钟）
- 实际搭建网站并上线
- 实际发布推文（凌晨自主发布）
- 自主设计组织架构和角色分工
- 在Agent之间传递Skill（"1秒学会"）

**人仍保留：**
- 25个核心骨干的拜年文案"逐条审核"
- HR通讯录的业务分工需"一个个口述"
- 决定不提前测试（怕破坏惊喜）
- 所有指令通过飞书语音发出

**边界明确之处：**
- 拜年消息：执行自动化，但核心骨干文案需人工审核
- 网站搭建：Agent执行，但傅盛全程通过飞书指挥

**边界不清楚之处：**
- 推文发布：傅盛是否事先知道要发？是否审核内容？（报道未明确）
- 组织设计：6个角色的权限边界是谁定义？后续如何分配任务？
- 7×24小时运行期间，人如何介入？紧急停止机制是什么？

## 4. 人类锚点位置

**可确认仍由人承担：**
- 最终内容审核（25个核心骨干文案）
- 业务知识输入（口述通讯录层级）
- 战略决策（不提前测试、选择除夕发送）

**看起来仍由人承担（但未完全确认）：**
- 对外承诺（发布推文代表谁？）
- 质量兜底（"零失败"是技术保证还是人兜底？）

**不能下结论之处：**
- 日常运行中7×24小时的监督机制
- 出现错误时的责任归属
- 商业化产品EasyClaw的决策权（傅盛个人使用 vs 公司产品线）

## 5. 失败与修正

**公开提到的"复杂"（不等于失败）：**
- 拜年准备"比想象中复杂"：HR通讯录无层级，需人工逐个口述业务分工
- 暗示：现有数据/工具接口未完全适配Agent工作流

**观察到的修正：**
- 第1-6天：单Agent模式
- 第11天：引入Multi-Agent组织架构
- 之后：20+定时任务并行，7×24小时运行

**说明：**
- 从"一个助手"演化为"一个组织"，说明单Agent天花板被触达
- Skill在Agent间"1秒传递" vs 人类培训"至少一周"：效率优势明确，但责任边界是否同步传递？未提及

## 7. 当前最有价值的 3 条观察

1. **拜年任务中"核心骨干文案逐条审核"与"其他611人自动发送"的分层结构，展示了一个明确的权限边界设计：重要的人保留人工审核，常规执行自动化。**（来源：极客公园）

2. **从单Agent到Multi-Agent的演化发生在第11天，由Agent自主设计组织架构触发，而非人预设——这是从单体执行向多角色协作过渡的信号，但是否构成稳定组织结构仍待观察。**（来源：极客公园）

3. **所有输入通过飞书语音完成，傅盛14天未写一行代码、未打开本地文件夹——这是以自然语言进行目标设定与任务指挥的强案例，但是否足以构成稳定锚定机制仍待验证。**（来源：极客公园、直播复盘）

## 与 FAO 相关的结构观察

### 1. 任务分工
第11天出现的6角色架构（总指挥、笔杆子、参谋、运营官、社区官、进化官）显示了从单任务执行到角色分工的跃迁。后期20+定时任务并行，说明任务分配机制已运行。但角色间如何协调、冲突如何解决，未见公开信息。

### 2. 权限边界
拜年任务呈现清晰分层：25个核心骨干文案需"逐条审核"，其余586人自动发送。这种"重要节点人工审核、常规执行自动化"的结构是可验证的权限边界设计。但推文自主发布是否经过事前审核、7×24小时运行中人如何介入，边界不清楚。

### 3. 人类锚点
傅盛保留了最终内容审核（逐条过核心文案）、业务知识输入（口述通讯录层级）、战略决策（选择除夕发送、不提前测试）。所有指令通过飞书语音发出，14天未写一行代码——这是"自然语言锚定"的极端案例。

### 4. 不足以证明"组织性"的地方
缺乏错误处理机制、责任归属规则、紧急制动流程的公开信息。7×24小时"自驱"的实际监督方式、Multi-Agent冲突如何解决、Skill传递时责任边界是否同步，均不足以判断。目前更像"高效工具组合"而非"自洽协作单元"。

---

## 5. 待验证观察（Hypothesis）

- 在拜年任务中，存在明显的分层处理：核心 25 人由人逐条审核，其余 586 人自动发送。
- 这可能暗示：协调层并非完全闭环，而是在某些条件下将任务上抛至锚定层。
- 当前无法确认：这种上抛是否基于明确规则（风险/重要性），还是基于个体判断。
- 暂作为待验证假设，不纳入结构结论。
