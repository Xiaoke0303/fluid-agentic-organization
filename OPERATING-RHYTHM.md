## 日常巡检（每日一次，5 分钟内）

| 优先级 | 动作 | 执行方式 |
|--------|------|----------|
| P0 | 新输入扫描（issue/discussion/PR） | 自动执行 |
| P0 | 标记需响应项（打标签、分类） | 自动执行 |
| P1 | 链接断裂检查（README、docs） | 自动执行，记录断裂点 |
| P1 | 版本号一致性检查 | 自动执行，记录差异 |
| P2 | 汇总需升级事项（高风险列表） | 自动执行，生成待确认清单 |

**执行原则**：
- 可直接执行 = 不改变责任锚点、不对外承诺、不修改治理
- 汇总后统一升级 = 减少人类维护者被打断次数

## 冻结项（本轮不自动化，或带边界执行）

| 原冻结项 | 修订后处理 | 边界 |
|----------|------------|------|
| issue 自动关闭 | 自动更新状态、补标签、生成关闭候选；不自动关闭涉及责任归属、理论争议、治理含义的 issue | 责任/争议/治理类必须升级 |
| discussion 自动回复 | 事实性指路回复可直接发送；涉及立场表达、价值判断、路线承诺的回复必须升级 | 立场/承诺/治理类必须升级 |
| PR 自动 merge | 完全冻结 | 改变代码现实，必须人类确认 |
| 状态报告自动生成 | 完全冻结 | 容易产生虚假完成感 |
| governance 自动修改 | 完全冻结 | 责任锚点变更，必须人类确认 |
| 版本号自动 bump | 完全冻结 | 对外承诺，必须人类确认 |

## 低风险可直接执行清单

| 类别 | 具体动作 | 边界条件 |
|------|----------|----------|
| 链接修复 | 修复 README/docs 中的断链 | 只改 URL，不改锚文本语义 |
| 版本号一致性 | 检查文档版本号与已存在 release/tag 的一致性 | 内部文档可自动修正；公开入口文件只生成修正建议或草稿 |
| 格式修正 | Markdown 格式、表格对齐、标题层级 | 不改变内容语义 |
| issue 分类 | 打标签、标记重复、初步分诊、生成关闭候选 | 不自动关闭责任/争议/治理类 issue |
| discussion 回复 | 事实性指路回复（引用现有文档、阅读顺序） | 立场/承诺/治理类必须升级 |
| changelog 事实记录 | 记录已发生的 commit、PR、修改 | 只记录历史，不预测未来 |
| PR 草稿补全 | checklist、模板填充、格式检查 | 不自动 merge |
| 状态标签更新 | 按证据更新标签；允许降级，必须记录原因 | 上调需证据，下调需记录证据变化 |
| 文档同步 | 多文件间的重复信息对齐 | 不改变任何一处的主张 |
| 拼写/语法修正 | 错别字、标点、句式 | 不改变术语定义 |

### issue 自动关闭候选限制

仅适用于重复、已失效、已由明确证据覆盖的问题。涉及责任、争议、治理、路线含义的 issue 不进入自动关闭候选。

### 状态标签状态机

```
上调（升级）：
[草稿] → [推断] → [已执行] → [已验证]
        ↑ 需证据          ↑ 需验证记录

下调（降级）：
[已验证] → [已执行] → [推断] → [草稿]
        ↓ 证据失效/被推翻  ↓ 证据不足  ↓ 待讨论
```

**降级必须记录**：原因 + 证据变化 + 时间。
