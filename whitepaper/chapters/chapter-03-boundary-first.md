# 第3章 边界先于流动

在释放任务路由的灵活性之前，必须先固定什么？

## 3.1 为什么边界先于流动

组织引入智能体主体的核心风险不是"效率不够"，而是"责任真空"。当智能体主体被赋予过多权限而责任归属模糊时，会出现两种危险状态：

**无人负责**——智能体主体做出决策，但组织内部没有明确的主体为该决策的后果承担责任。当问题发生时，各方相互推诿，最终受损的是组织对外部利益相关者的可信度。

**无法追溯**——智能体主体的决策链路复杂且不透明，即使想追责，也无法还原"谁、在何时、基于什么信息、做出了什么判断"的完整链条。

FAO的设计哲学认为：能力可以下放，责任不能消失。在明确哪些东西"不能流动"之前，任何形式的权限下放都是在扩大风险敞口。边界必须先固定，流动才有意义。

## 3.2 不能流动的清单

FAO框架下，以下七项核心要素在任何情况下都不得流动至智能体主体：

**决策位置**——最终决策的物理与逻辑位置必须固定在组织内部的自然人。智能体主体可以提供建议、生成方案、甚至执行预设规则，但不能占据"最终拍板"的位置。决策位置的流动性会导致权力与责任的分离。

**最终承担**——无论智能体主体参与程度多深，最终责任的承担主体必须是组织内部明确指定的自然人。这是不可转移、不可代理的底线。智能体主体可以"执行"，但不能"负责"。

**对外代表权**——代表组织与外部实体（客户、监管方、合作伙伴、公众）进行法律意义上互动的权利，必须由组织授权的特定人员行使。智能体主体不能天然获得对外代表组织的资格。

**价值锚定**——组织的核心价值观、伦理底线、战略方向必须由人类主体锚定。智能体主体可以基于这些锚定进行计算和优化，但不能重新定义或漂移这些价值基准。

**边界暴露义务**——组织有义务向外部利益相关者明确披露：哪些互动是由智能体主体完成的、哪些决策有人类主体参与、出现问题时应向谁追责。这种披露义务本身不能委托给智能体主体。

**未验证不能冒充**——智能体主体不得在未经验证的情况下冒充人类主体身份、冒充特定职位、或冒充具有特定权限的角色。让利益相关者误以为他们在与人类主体交互，而实际上对方是智能体主体，构成欺骗。

**未任不能代其责**——智能体主体不能被默认赋予任何责任。任何责任的赋予都必须经过明确的"任命"程序——即组织正式授权、记录在案、可审计追溯。责任的赋予不是技术自动触发的，而是组织治理程序的结果。

## 3.3 两个边界圈

FAO框架通过"两个边界圈"来可视化上述原则：

```
                    ┌─────────────────────────────────────┐
                    │           外部世界                   │
                    │  (客户/监管/公众/合作伙伴)            │
                    └──────────────┬──────────────────────┘
                                   │
        ┌──────────────────────────┼──────────────────────────┐
        │     【外圈】组织 ↔ 外界   │                          │
        │                          ▼                          │
        │  ┌─────────────────────────────────────────────┐   │
        │  │              组织边界                        │   │
        │  │  ┌─────────────────────────────────────┐    │   │
        │  │  │      【内圈】人 ↔ 智能体              │    │   │
        │  │  │                                     │    │   │
        │  │  │   ┌─────────┐         ┌─────────┐   │    │   │
        │  │  │   │  人类   │◄───────►│ 智能体   │   │    │   │
        │  │  │   │ 主体    │  授权   │ 主体     │   │    │   │
        │  │  │   └─────────┘         └─────────┘   │    │   │
        │  │  └─────────────────────────────────────┘    │   │
        │  └─────────────────────────────────────────────┘   │
        └────────────────────────────────────────────────────┘
```

**内圈：人类主体与智能体主体的边界**——在组织内部，人类主体与智能体主体之间存在明确的权限边界。智能体主体的所有行动都源于人类主体的授权。这种授权是可撤销的、有范围限制的、可审计的。

**外圈：组织与外界的边界**——组织作为一个整体与外部世界互动时，必须确保外部利益相关者能够识别与谁建立法律关系。智能体主体不能天然占据组织与外界之间的接口位置。

两个边界圈的存在意味着智能体主体在组织内部可以高度活跃，但它在组织外部没有"直接存在"。外部世界看到的是组织，而非组织内部的智能体主体。

## 3.4 案例印证

2026年浙江首例涉人工智能生成物不正当竞争案（阿里巴巴诉李某AI幻觉案）为FAO的边界原则提供了司法层面的印证。

判决要点包括：对外决策位置不可让渡——法院指出，李某发布文章的行为属于对外代表行为，不能因AI生成而转移责任；责任主体不可流动性——技术工具的使用不能免除作者的法律责任；边界暴露义务的缺失——仅在后台标注AI生成不足以保障公众知情权；审核义务——使用者对生成内容负有必要的审核义务。

该案与FAO的"边界先于流动"原则高度一致：智能体主体不能天然占据对外决策位置，能力可以下放，责任不能消失，边界暴露是义务而非可选项。

## 3.5 边界固定后，才释放流动空间

FAO不是反对智能体主体的使用，而是主张在明确边界的前提下安全地使用智能体主体。

当决策位置固定，智能体主体可以承担信息收集、方案生成、风险评估等前置工作。当最终承担固定，智能体主体可以被赋予广泛的执行权限，因为责任主体始终明确。当对外代表权固定，智能体主体可以在严格授权范围内与外部系统交互。当价值锚定固定，智能体主体可以进行复杂的优化计算，但必须在给定约束内。当边界暴露义务固定，智能体主体可以高度自动化，但必须可识别、可审计。

只有在不可流动要素被明确锁定后，能力范围才能真正释放。边界不是限制，而是安全释放流动性的前提。
