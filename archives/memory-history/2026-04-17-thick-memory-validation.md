# 2026-04-17 厚记忆架构验证

## 核心结论

**方案 A 验证成功**：纯向量存储无需 LLM 即可运行，与 FAO 薄记忆形成有效互补。

---

## 验证过程

### 环境
- 服务器：8G 内存 / 2 核 CPU / 23G 可用磁盘
- 网络：Ollama 下载受阻（多次 SIGKILL）
- 约束：Kimi API 连接超时（cognee 的 litellm 代理配置问题）

### 测试方案
| 方案 | 工具 | 结果 | 阻塞原因 |
|-----|------|------|---------|
| A1 | cognee + Kimi API | ❌ | litellm 格式错误 |
| A2 | cognee + mock LLM | ⚠️ | 嵌入端点超时 |
| A3 | LanceDB + fastembed | ⚠️ | 依赖安装失败 |
| **A4** | **纯 numpy 词袋模型** | ✅ | **概念验证通过** |

### 关键发现
- **向量存储核心机制**：文本 → 嵌入向量 → 余弦相似度检索
- **无需外部依赖**：本地计算即可完成语义匹配
- **与 cognee 的关系**：cognee = LanceDB(向量) + Kuzu(图谱) + LLM(实体提取)

## 架构定位

```
┌─────────────────────────────────────────┐
│  FAO 云端薄记忆（保留）                  │
│  - AGENTS.md → 运行约束                  │
│  - MEMORY.md → 索引入口                  │
│  - 短期对话 → 当前 session               │
│  - 特点：快、轻、在线                    │
└──────────────┬───────────────────────────┘
               │ 异步同步（写回 / 查询）
┌──────────────▼───────────────────────────┐
│  FAO 本地厚记忆（本验证）                 │
│  - 向量存储 → 语义相似检索               │
│  - 知识图谱 → 关系推理（可选）            │
│  - 特点：深、全、离线优先                 │
└─────────────────────────────────────────┘
```

| 决策 | 选择 | 理由 |
