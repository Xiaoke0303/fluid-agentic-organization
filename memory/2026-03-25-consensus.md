# 今天的主线和共识

2025-03-25

## 核心转变

FAO 不是在解决"五层"的问题。
五层最多是一个分析镜头，不是 FAO 的本体。

FAO 更像一种协作方法论——当组织中的代理从"只有人"变成"人 + 智能体"以后，任务怎么分配，判断怎么支持，责任怎么锚定。

## 两种主体

**人类主体**
- 局限：带宽窄、计算慢、注意力稀缺、易疲劳
- 保留：最终承担、价值锚定、对外代表、合法性位置

**智能体主体**
- 局限：容易伪完成、越界、把推断当事实、不能天然承担后果
- 擅长：高带宽、高计算、持续处理、并行推进、数据加工与判断支持

## FAO 的灵魂：路由

不是普通 workflow 路由，而是：
- 谁先承接哪一段任务
- 哪些信息加工可以流动
- 哪些判断支持可以流动
- 什么时候上抛
- 什么地方必须锚定回人

**核心句**：能力可以下放，责任不能消失。

## 不是先定义什么流动，而是先定义什么不能流动

**不能流动的**：
- 智能体不能天然占据会对外部环境产生后果的决策位置
- 最终承担不能流动到不能承担后果的主体
- 对外代表权不能自然下放给智能体
- 边界暴露义务不能消失
- 未验证不能冒充已确认
- 未任不能代其责

**两个边界圈**：
- 组织内的人 ↔ 外界
- 组织 ↔ 外界

智能体并不能天然占据这两个边界位置。

## 代理问题没有消失，只是被改写

FAO 不能取消代理问题，但可能缓解其中由人类带宽和中介摩擦造成的部分。

## 关键工作句

1. FAO 不是在解决五层的问题。
2. FAO 是一种在人类主体与智能体主体共存条件下，围绕任务路由与责任锚定展开的协作方法论。
3. 两种主体各有局限。
4. FAO 不是先定义什么流动，而是先定义什么不能流动。
5. 对外部环境产生后果的决策位置，不能流动到不能承担后果的主体。
6. FAO 的灵魂不是层级，而是路由。
7. 智能体不能取消代理问题，但可能缓解其中由人类带宽与中介摩擦造成的部分。

## Soul 收口

- 不争、不夺、不僭、不硬
- 知止、不住相、守中

更深：名实不乱、位责不僭。有核而应变，不因外物而失其位。
